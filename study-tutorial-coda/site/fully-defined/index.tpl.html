Under- and overconstraining

When adding constraints and parametric behavior to 3D CAD models, the goal is to add exactly the right amount of constraints to the model. Adding too little constraints results in an under-constrained model, while adding too many constraints will result in an over-constrained model. When having exactly the right amount of constraints, the model is fully-constrained or fully-defined

In Autodesk Fusion 360, the 3D CAD modeling package we will be using throughout this study, each 2D sketches can be in one of the following states:

  1. Under-constrained: some entities in the sketch are constrained, but not all. Some entities are still able to move freely, while others are not. These undesired "degrees-of-freedom" can lead to unexpected behavior in the model when making parametric changes.

    In Fusion 360, an under-constrained sketch can be identified in the content browser as shown below. Note the "edit" icon next to the sketch name, indicating the sketch is still a "work-in-progress" as not all constraints are added yet.

    When editing a sketch in Fusion 360, you can also observe the state of individual entities through their color. More specifically, unconstrained entities have a blue color when not selected, as shown below.

  2. Fully-constrained/fully-defined: all possible and relevant constraints and dimensions are applied to the sketch. The sketch can only be modified by changing the values of the dimensional constraints, and there will be no unexpected behavior when making changes.

    In Fusion 360, a fully-constrained/fully-defined sketch can be identified in the content browser as shown below. Note the "lock" icon next to the sketch name, indicating all entities in the sketch are locked through constraints.

    When editing a sketch in Fusion 360, you can also observe the state of the individual entities through their color. More specifically, fully-constrained entities have a black color when not selected, as shown below.

  3. Over-constrained: there are duplicate constraints in the model, enforcing the same relation in different ways. Fusion 360 prevents this state from occurring by showing an error and removing the inserted constraint when this conflict occurs. The error message is shown below.

    To understand why this error is shown, consider the simplistic example shown below. As the bottom line is already perpendicular to a line that is vertically constrained, it is already implied that the bottom line will always be horizontal. Attempting to add a horizontal constraint thus results in a duplicate specification of the same relation, which is not possible.

In conclusion, if in Autodesk Fusion 360 every sketch of a model is fully defined/fully constrained, then the model itself is fully defined/fully constrained, resulting in predictable parametric behavior and a robust model.
